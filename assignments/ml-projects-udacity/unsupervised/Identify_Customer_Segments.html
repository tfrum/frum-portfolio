def convert_to_nan(df, summary_df):
    for _, row in summary_df.iterrows():
        attribute = row['attribute']
        missing_values = row['missing_or_unknown']
        # Parse the string of missing values to a list of values
        if isinstance(missing_values, str):
            missing_values = missing_values.strip('[]').replace(' ', '').split(',')
            missing_values = [
                int(value) if value.lstrip('-').isdigit() else value.replace('\'', '').strip()
                for value in missing_values
            ]
        if missing_values != [''] and attribute in df.columns:
            df[attribute] = df[attribute].replace(missing_values, np.nan)
    return df

def calculate_sse(data, cluster_range):
    sse = []
    for k in cluster_range:
        kmeans = KMeans(n_clusters=k, random_state=42)
        kmeans.fit(data)
        sse.append(np.abs(kmeans.score(data)))
    return sse

Step 2.3: Interpret Principal Components¶

Now that we have our transformed principal components, it's a nice idea to check out the weight of each variable on the first few components to see if they can be interpreted in some fashion.

As a reminder, each principal component is a unit vector that points in the direction of highest variance (after accounting for the variance captured by earlier principal components). The further a weight is from zero, the more the principal component is in the direction of the corresponding feature. If two features have large weights of the same sign (both positive or both negative), then increases in one tend expect to be associated with increases in the other. To contrast, features with different signs can be expected to show a negative correlation: increases in one variable should result in a decrease in the other.

  • To investigate the features, you should map each weight to their corresponding feature name, then sort the features according to weight. The most interesting features for each principal component, then, will be those at the beginning and end of the sorted list. Discussion 3.3: Compare Customer Data to Demographics Data¶

So we could focus on a number of options here, naturally. Cluster 2 is the natural option, as the customer data has a severe overrepresentation there compared to any other category. That cluster represents a few key factors, mostly gender distribution and social-psychological traits.

And much to my shock, cluster 2 is nearly all males. There's 1 female entry in all 40147 members of that cluster. Wow. That's just, statistically shocking. And I verified it multiple times. It seems that the other contributing factors are correlated enough with being male that it created essentially a male-only cluster.

It would seem that the customers the company is most overrepresented with are older males who are financially savvy. That this group clusters so tightly around gender is notable, and represents, probably, an overreliance on a demographic that's shrinking and that probably isn't all that likely to spend frivolously as they age toward retirement. There's a lot of implications there, naturally.

Regarding the subset of data that had high missingness. I approached that in two ways.

  • A: We dropped columns with high missingness and then all rows with missing data, resulting in minimal loss of rows (<3%)
  • B: We kept those columns and dropped all rows with missing data, resulting in a larger 20-25% drop in the number of rows.

Likewise, we ran a correlation analysis on the high missingness rows and found that they all had significant correlation with eachother, indicating that it was likely an issue of data quality. A distinct group you would expect to see patterns in that many rows, and while there were hotspots on the correlation matrix, there were few, if any, cold spots. When missing data correlates with missing data broadly, it's just bad data.
